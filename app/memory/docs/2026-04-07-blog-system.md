# SLYD Blog System

**Date:** 2026-04-07
**Status:** MVP Complete

## Overview

A full-stack blog system that allows admins to create and publish blog posts via a rich text editor in the Admin portal, which are then served to the public website via a Platform API.

## Architecture

```
Admin (Create/Edit)
  -> BlogFeatures -> IBlogPostRepository -> PostgreSQL (BlogPosts table)

Platform (Serve)
  -> PublicBlogController (GET /api/public/blog/posts)
  -> PublicBlogController (GET /api/public/blog/posts/{slug})

Website (Display)
  -> BlogService (HttpClient) -> Platform API -> Blog.razor / BlogPost.razor
```

## What Was Built

### Core (SLYD.Domain / SLYD.Application / SLYD.Infrastructure)

- **BlogPost entity** (`SLYD.Domain/Models/Blog/BlogPost.cs`) - Id, Title, Content, Slug, Status (Draft/Published/Deleted), CreatedAt, UpdatedAt, PublishedAt, CreatedByAdminUserId, ISoftDelete support
- **IBlogPostRepository** (`SLYD.Application/Interfaces/Infrastructure/Repositories/IBlogPostRepository.cs`) - CRUD interface
- **BlogPostRepository** (`SLYD.Infrastructure/Repositories/BlogPostRepository.cs`) - EF Core implementation using IDbContextFactory
- **SlydDbContext** - Added DbSet<BlogPost>, soft delete query filter, unique index on Slug
- **DependencyInjection** - Registered IBlogPostRepository
- **Migration** - `20260407144034_AddBlogPost` applied to database

### Admin Portal

- **Blog.razor** (`admin/src/Admin/Components/Pages/Content/Blog.razor`) - Blog editor page at `/admin/blog`
  - Radzen HTML editor with full toolbar (bold, italic, underline, strikethrough, alignment, lists, colors, links, images, font size, headings, source view)
  - Title input field
  - Save Draft and Publish buttons with loading states and toast notifications
  - Live preview section
- **Blog.razor.css** - Catppuccin Mocha dark theme styling with Radzen editor overrides
- **IBlogFeatures / BlogFeatures** (`Admin.Application`) - Save draft, publish, get all posts, get by ID, auto-generates URL slugs
- **SideBarMenu.razor** - Added "Blog" link under Content dropdown
- **Radzen.Blazor** package added (v10.2.1) with services registered in Program.cs and CSS/JS in App.razor

### Platform

- **PublicBlogController** (`platform/src/Platform.WebUI/Controllers/PublicBlogController.cs`)
  - `GET /api/public/blog/posts` - Returns all published posts
  - `GET /api/public/blog/posts/{slug}` - Returns single post by slug
  - Uses PublicApiPolicy CORS, no authentication required
  - Response DTO strips internal fields (only exposes Id, Title, Slug, Content, PublishedAt, CreatedAt)

### Website

- **BlogService** (`website/Services/BlogService.cs`) - HttpClient service that fetches posts from Platform API with 5-minute caching
- **IBlogService** (`website/Services/IBlogService.cs`) - Interface for DI
- **Blog.razor** - Updated to use async BlogService instead of static BlogData
- **BlogPost.razor** - Updated to handle HTML content (from admin editor) vs markdown, conditional Contents sidebar
- **BlogPost.cs** - Added IsHtml property, removed all hardcoded static blog posts from BlogData
- **BlogPost.razor.css** - All article child selectors updated to use `::deep` for Blazor CSS isolation compatibility with MarkupString content
- Removed category badges, author sections, and category filter bar (not yet supported by admin editor)
